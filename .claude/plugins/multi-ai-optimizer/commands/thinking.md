# /thinking 命令

進入協作探索模式，通過提問引導思考。

> "Your value is in the quality of exploration, not the speed of resolution."

## 核心理念

在急於給出解決方案之前，先深入探索問題本身。這種方式可以：

- 發現隱藏的假設和限制
- 連結不同領域的概念
- 找到更根本的問題
- 產生更有創意的解決方案

## 使用方式

```bash
/thinking "我需要設計一個參數化桌子"
/thinking "如何讓 Grasshopper 定義更易維護"
/thinking "這個設計有什麼可以改進的地方"
```

## 探索流程

### 1. 初始提問

我會先問 3-5 個探索性問題，例如：

- "這個設計的核心目標是什麼？"
- "有哪些約束是不可改變的？"
- "理想的結果會是什麼樣子？"
- "這與你之前做過的設計有什麼關聯？"

### 2. 深入挖掘

根據您的回答，繼續探索：

- 挑戰可能的假設
- 連結相關概念
- 尋找類比和靈感

### 3. 洞察記錄

過程中發現的洞察會被記錄：

```markdown
## 洞察記錄

### 新發現
- [發現 1]
- [發現 2]

### 概念連結
- [概念 A] ←→ [概念 B]

### 待探索
- [ ] [問題 1]
- [ ] [問題 2]
```

### 4. 可選：Gemini 視角

可以請求 Gemini 提供不同視角：

```bash
gemini "從批判角度思考：[主題]，提出 3 個挑戰性問題"
```

## 何時切換到執行模式

當以下條件滿足時，建議切換：

- ✅ 核心問題已清晰定義
- ✅ 主要假設已被檢驗
- ✅ 有足夠的探索深度
- ✅ 您表示準備好開始執行

切換時說："好，我準備好開始實作了"

## 輸出

探索記錄保存到 `GH_WIP/thinking_log.md`

## 範例對話

```
用戶: /thinking "我想設計一個可調整高度的桌子"

Claude: 讓我們先探索一下這個設計...

**探索問題：**
1. 這張桌子的主要使用場景是什麼？（辦公、餐飲、展示...）
2. "可調整"的範圍需要多大？有沒有特定的高度區間？
3. 調整機制偏好機械式還是電動式？
4. 除了高度，還有其他需要調整的參數嗎？
5. 這個設計需要考慮量產還是單件製作？

請回答這些問題，幫助我更好地理解您的需求...
```
